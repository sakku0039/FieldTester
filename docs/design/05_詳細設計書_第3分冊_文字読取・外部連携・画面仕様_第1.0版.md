# 05_詳細設計書_第3分冊_文字読取・外部連携・画面仕様

版数: 第1.0版

## 本書の位置づけ

本書は、共通基本原則に従い、OCR対象、確認フロー、画面構成、外部連携を定義する。OCR確認結果はProvider非依存のクラウドAPIとSync Agentを経由し、出荷DB内の`FieldTest.TestResult`および`FieldTest.OcrResult`へ登録する。

## 1. OCR対象

OCRは、フレッシュ試験結果の確認補助として、黒板写真または試験結果写真から値候補を抽出する。抽出対象はスランプ、フロー、空気量、コンクリート温度、単位容積質量、塩化物量、外気温、単位水量、打設箇所、備考などとする。

## 2. 確認フロー

1. 利用者がOCR対象写真を登録する。
2. クラウドAPIがOCRを実行し、値候補と信頼度を返す。
3. 利用者が確認画面で値を修正・確定する。
4. 確定結果はクラウドに保存され、Sync Agentへ同期される。
5. Sync AgentはProviderにかかわらず`FieldTest.TestResult`と`FieldTest.OcrResult`へ登録する。

## 3. 画面構成

|画面|内容|
|---|---|
|ログイン|利用者認証と権限確認。|
|出荷予定一覧|予定日、現場、出荷予定、状態で検索する。|
|出荷実績詳細|出荷実績、配合、現場、試験登録状態を表示する。|
|写真登録|OCR対象写真、記録写真を登録する。|
|OCR確認|候補値、信頼度、手修正欄を表示する。|
|試験結果入力|フレッシュ試験値を入力・確定する。|
|保存完了|登録結果、同期状態、エラーがあれば再試行情報を表示する。|

## 4. 外部連携

OCRエンジン、写真ストレージ、クラウドAPIはProvider非依存とする。Provider固有処理はSync AgentのAdapterとDB Migrationに限定する。


## 7. 黒板・TP・品質DBとの境界

第1.0版OCR同期は、撮影済み写真からフレッシュ試験値候補を抽出し、利用者が確認した値を`FieldTest.TestResult`および`FieldTest.OcrResult`へ登録する処理である。黒板テンプレート生成、黒板レイアウトDB読取、TP採取、供試体、強度試験、品質帳票DBへの登録は第1.0版の同期処理に含めない。

## 8. 値の保持形式

OCR元文字列は`*_text`、確定数値は`*_value`として保持する。数値化できない候補は`*_text`に保持し、`*_value`はNULLとする。画面では元文字列、補正後数値、信頼度、確認者、確認日時を表示できるようにする。
