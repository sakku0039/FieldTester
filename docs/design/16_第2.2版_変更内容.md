# 現場試験アプリ 設計資料 第2.2版 変更内容

## 1. 用語の修正

- クラウド環境の名称を「現場試験アプリ用クラウド環境」へ統一。
- 技術説明では「クラウド側」「アプリ用クラウドDB」「クラウド連携API」を使用。
## 2. ローカルデータベース配置の確定

- 新しい物理DBを作成しない方針に統一。
- 既存の出荷管理DBを利用。
- 既存業務テーブルは dbo スキーマを維持。
- 追加12テーブルを FieldTest スキーマへ配置。
- データベース名とスキーマ名を混同する表現を廃止。
## 3. 設定XMLとの関係

- LibertyDatabaseSetting.xml に新規DB名項目を追加しない。
- 既存の出荷管理DB名設定を利用。
- スキーマ名は Sync Agent 設定 LocalIntegrationSchemaName（初期値 FieldTest）で管理。
- 起動時にスキーマ、必要テーブル、権限を検証。
## 4. DDL・SQLの修正

- 追加テーブルを [FieldTest] スキーマで作成。
- スキーマ作成、存在確認、権限確認のSQLを追加。
- 参照例を [データベース].[スキーマ].[テーブル] 形式へ統一。
## 5. 要件・受入条件の追加

- DB配置および設定XML方針に関する要件を追加。
- 受入条件A-121〜A-123を追加。
- 管理表、基本設計、詳細設計、説明資料を同期更新。
## 6. 図の追加

- データベース配置・設定解決図を追加。
- システム全体構成図のクラウド側表現を修正。
## 7. 版・ファイル名

- 全資料を第2.2版（完成版）へ更新。
- ファイル名は日本語表記に統一。

## 8. Markdown対応パッケージ

- 資料体系、要件定義、基本設計、詳細設計5分冊をMarkdownへ変換しました。
- 文書内の図と画面イメージは `画像` フォルダへ抽出し、相対パスで参照します。
- 追跡管理表、説明資料、DDL、図版、参考PDFは用途上適切な既存形式を維持します。
- 内容上の版は第2.2版のままで、今回の変更は配布形式の最適化です。
