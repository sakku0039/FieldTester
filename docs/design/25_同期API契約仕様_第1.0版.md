# 25_同期API契約仕様

版数: 第1.0版

## 本書の位置づけ

本書は、クラウドAPIとSync Agentの同期API契約を定義する。Provider差異はSync Agent Adapterが吸収し、API payloadはProvider非依存のDTOを基本とする。

## 1. 共通仕様

|項目|仕様|
|---|---|
|認証|Agent credentialによるBearer token。短命token、audience、scope、tenantId、orgId、plantIdを検証する。|
|冪等キー|全POSTに`eventId`または`idempotencyKey`を必須とする。|
|correlation|`correlationId`を全APIで受け渡す。|
|日時|UTC ISO-8601。|
|失敗時|4xxは入力・認可・設定エラー、5xxは再試行対象。|
|ACK|Sync AgentがFieldTestへ永続化し、再処理可能な状態を記録した後にACKする。|

## 2. API一覧

|Method|Path|方向|概要|冪等性|
|---|---|---|---|---|
|GET|`/agent/config`|Cloud -> Agent|Agent設定、同期周期、更新manifest参照を取得|なし|
|POST|`/agent/heartbeat`|Agent -> Cloud|起動状態、Provider診断、DB診断、Migration状態を通知|`agentId + timestamp`|
|POST|`/shipments/import`|Agent -> Cloud|Provider既存DBから読取った予定・出荷・現場DTOを送信|`source_shipment_key_hash_sha256`|
|GET|`/test-results/export?since=`|Cloud -> Agent|確定済み試験結果イベントを取得|eventId|
|POST|`/test-results/{eventId}/ack`|Agent -> Cloud|試験結果イベントの永続化完了を通知|eventId|
|GET|`/photos/export?since=`|Cloud -> Agent|写真メタデータイベントを取得|eventId|
|POST|`/photos/{eventId}/ack`|Agent -> Cloud|写真メタデータ永続化完了を通知|eventId|
|GET|`/ocr-results/export?since=`|Cloud -> Agent|OCR確認結果イベントを取得|eventId|
|POST|`/ocr-results/{eventId}/ack`|Agent -> Cloud|OCR確認結果永続化完了を通知|eventId|
|POST|`/audit-events/import`|Agent -> Cloud|監査・診断イベントを送信|eventId|
|GET|`/agent/update-manifest`|Agent -> Cloud|自動更新manifestを取得|なし|

## 4. ACK条件

ACKは、対象イベントが`FieldTest.InboundEvent`へ記録され、対応する`FieldTest.TestResult`、`FieldTest.TestPhoto`、`FieldTest.OcrResult`へ登録済み、または同一自然キーで登録済みであることを確認してから送信する。DB登録前のACKは禁止する。

## 5. 再試行

再試行は指数バックオフとし、`retry_count`、`next_retry_at`、`last_error_code`、`last_error_message`を`InboundEvent`または`OutboxEvent`へ記録する。恒久エラーは`DeadLetter`へ移行し、監査イベントを送信する。
