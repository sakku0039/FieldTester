# 21_実装前設定確認チェックリスト

版数: 第1.0版

## 本書の位置づけ

本書は、実装前に確認するProvider設定、DB名、権限、Migration、非使用DB、参照アプリ連携のチェック項目を定義する。共通基本原則は上位資料を正とし、本書では実施可能な確認項目に落とし込む。

## 1. 共通確認

|ID|対象|確認内容|
|---|---|---|
|C-001|Provider|Sync Agent設定で`Labonity`または`Liberty`が明示選択されている。|
|C-002|入口設定ファイル|選択Providerに対応する入口設定ファイルの完全パスがSync Agent設定に登録されている。|
|C-003|自動切替禁止|設定不備時に別Providerへ自動切替しない。|
|C-004|秘密値|SQLパスワード等がログや監査に平文出力されない。|
|C-005|plantId / plantCode|plantIdはGUID、plantCodeはProvider側工場コードとして分離されている。|
|C-006|FieldTest|解決した出荷DB内に`FieldTest` Migrationを適用できる。|

## 2. Labonity確認

|ID|対象|確認内容|
|---|---|---|
|L-001|LibLocal.xmlパス|Sync Agent設定の`labonity.libLocalXmlPath`が存在し、読み取れる。|
|L-002|サーバーフォルダ|`LibLocal.xml`の`p_サーバーフォルダパス`が取得できる。|
|L-003|LibertyDatabaseSetting.xml|サーバーフォルダ配下で解決でき、読み取れる。|
|L-004|必須DB名|SQL Server名、マスターDB、出荷管理DB、共通設定DBが取得できる。SQL Server名が空欄の場合は失敗。|
|L-005|非使用DB|品質管理DB名が存在しても、第1.0版OCR同期の必須接続先にならない。|
|L-006|登録|出荷管理DBの`FieldTest.TestResult` / `FieldTest.TestPhoto` / `FieldTest.OcrResult`へ登録できる。|

## 3. Liberty確認

|ID|対象|確認内容|
|---|---|---|
|R-001|LIBLocal.Iniパス|Sync Agent設定の`liberty.libLocalIniPath`が存在し、CP932で読み取れる。|
|R-002|サーバーパス|`LIBLocal.Ini`の`[端末設定] サーバーパス`が取得できる。|
|R-003|LIBCTRL.ini|サーバーパス配下で解決でき、CP932で読み取れる。|
|R-004|必須DB名|SQL Server名、出荷データDB、共通マスタDB、初期工場コードが取得できる。SQL Server名が空欄の場合は失敗。|
|R-005|非使用DB|品管データDB、品管データ2DB、写真台帳DBが存在しても、第1.0版OCR同期の必須接続先にならない。|
|R-006|登録|出荷データDBの`FieldTest.TestResult` / `FieldTest.TestPhoto` / `FieldTest.OcrResult`へ登録できる。|
|R-007|Providerアプリ|Liberty側アプリケーションが`FieldTest`スキーマを参照できる。|

## 4. あると安全な確認データ

実装前には、予定、出荷、現場の代表データ各1件、写真1件、OCR結果1件、同一eventId再送試験データ1件があると、DTO変換と冪等性の最終確認ができる。


## 5. 追加確認

|ID|対象|確認内容|
|---|---|---|
|C-007|source key|同じ既存出荷から常に同じcanonical JSONとSHA-256が生成される。|
|C-008|source result key|同じ試験結果から常に同じcanonical JSONとSHA-256が生成される。|
|C-009|写真重複|同一結果・同一写真種別・同一SHA-256を重複登録しない。|
|C-010|Inbound/Outbox|通信断後の再試行で処理状態が復旧できる。|
|C-011|参照View|Providerアプリ用Viewが期待列を返す。|
|C-012|Migration|互換不能な既存FieldTestオブジェクトがある場合に停止する。|
