# 現場試験アプリ 設計資料一式 第2.2版（完成版・Markdown対応）

本資料一式は、現場試験アプリ、現場試験アプリ用クラウド環境、Sync Agent、既存Labonityとの連携について、要件から詳細仕様・受入条件までを整理したものです。

## 推奨する確認順序

1. `10_ステークホルダー説明資料_第2.2版.pptx`
2. `01_要件定義書_第2.2版.md`
3. `02_基本設計書_第2.2版.md`
4. `03`〜`07`の詳細設計書（Markdown）
5. `08_要件・受入条件・追跡管理表_第2.2版.xlsx`
6. `09_現場試験アプリ_ローカル連携テーブル定義_第2.2版.sql`

## ファイル形式の方針

| 資料 | 形式 | 理由 |
|---|---|---|
| 資料体系、要件定義、基本設計、詳細設計 | Markdown | 検索、差分確認、レビュー、版管理に適しているため。 |
| ステークホルダー説明資料 | PowerPoint | 会議投影と説明進行に適しているため。 |
| 要件・受入条件・追跡管理表 | Excel | フィルター、状態、担当、証跡を管理するため。 |
| ローカル連携テーブル定義 | SQL | 実行・レビュー可能なDDLとして保持するため。 |
| 構成図・フロー図 | PNG | 単独で閲覧・貼付できる図版として保持するため。 |
| 既存テーブル定義 | PDF | 元資料の複雑な表組みとページ構成を維持するため。 |

## 第2.2版で確定したローカルデータベース方針

- 新しい物理データベースは作成しません。
- 既存の出荷管理データベース（通常は `NaDat`）を利用します。
- 既存業務テーブルは `dbo` スキーマのまま変更しません。
- 現場試験アプリ連携用の追加12テーブルは `FieldTest` スキーマに配置します。
- `LibertyDatabaseSetting.xml` には新しいデータベース名項目を追加しません。
- `FieldTest` は Sync Agent 側のスキーマ設定として扱い、起動時に存在・テーブル・権限を検証します。
