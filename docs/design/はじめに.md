## ファイル形式の方針

| 資料 | 形式 | 理由 |
|---|---|---|
| 資料体系、要件定義、基本設計、詳細設計 | Markdown | 検索、差分確認、レビュー、版管理に適しているため。 |

| 既存テーブル定義 | PDF | 元資料の複雑な表組みとページ構成を維持するため。 |

## 第2.2版で確定したローカルデータベース方針

- 新しい物理データベースは作成しません。
- 既存の出荷管理データベース（通常は `NaDat`）を利用します。
- 既存業務テーブルは `dbo` スキーマのまま変更しません。
- 現場試験アプリ連携用の追加12テーブルは `FieldTest` スキーマに配置します。
- `LibertyDatabaseSetting.xml` には新しいデータベース名項目を追加しません。
- `FieldTest` は Sync Agent 側のスキーマ設定として扱い、起動時に存在・テーブル・権限を検証します。
