# 23_OCR同期対象DB・非使用DB設計

版数: 第1.0版

## 本書の位置づけ

本書は、第1.0版OCR同期で使用するDBと使用しないDBを定義する。同期対象は出荷DBとマスターDBに限定し、結果登録は出荷DB内の`FieldTest`スキーマで完結する。品質系DBは将来拡張の範囲とする。

## 1. 第1.0版の同期範囲

第1.0版は、出荷予定・出荷実績・現場情報を基に、フレッシュ試験結果、写真、OCR確認結果を登録する。同期対象DBは、Providerごとに出荷DBとマスターDBへ限定する。

|Provider|必須読取DB|必須登録DB|代表DB名|
|---|---|---|---|
|Labonity|マスターDB、出荷管理DB|出荷管理DBの`FieldTest`スキーマ|MSDAT、NADAT|
|Liberty|共通マスタDB、出荷データDB|出荷データDBの`FieldTest`スキーマ|LMASTER、LSYUDAT|

## 2. 品質DBを同期対象外にする理由

OCR確認結果は、フレッシュ試験結果の各値を登録する処理であり、登録先は両Providerとも出荷DB内の`FieldTest`スキーマで完結する。TP採取、供試体、強度試験、品質帳票DBへの登録は第1.0版の同期処理に含めない。このため、品質DBは設定ファイルに存在しても必須接続先にしない。

## 3. Provider別の非使用DB

|Provider|設定ファイルに存在し得る非使用DB|第1.0版の扱い|
|---|---|---|
|Labonity|品質管理DB、販売管理DB、動荷重DB|読取・登録・接続診断の必須対象にしない。|
|Liberty|品管データDB、品管データ2DB、写真台帳DB、販売DB、受払DB、試験場DB|読取・登録・接続診断の必須対象にしない。|

## 4. 将来拡張との境界

将来、TP採取、供試体、強度試験、品質帳票まで同期対象を広げる場合は、品質DBを別機能として有効化する。その場合でも、Provider設定の入口ファイルパス解決、Provider固定、秘密値保護、監査、FieldTest登録の原則は本設計を継続する。


## 5. 黒板・TP採取・品質機能の扱い

第1.0版OCR同期では、黒板テンプレート生成、黒板レイアウトDB読取、TP採取結果入力、供試体、強度試験、品質帳票DBへの登録は扱わない。OCR対象は、現場アプリで登録された写真と、利用者が確認したフレッシュ試験値である。将来、TP採取や強度試験へ拡張する場合は、品質DBを別機能として明示的に追加する。
