# 16_第1版確定事項・構成方針

版数: 第1.0版

## 本書の位置づけ

本書は、第1.0版として確定する構成方針を整理する。共通基本原則は`00_資料体系・レビューガイド_第1.0版.md`および`99_統合設計仕様書_第1.0版.md`を正とし、本書では確定事項と採用構成を要約する。

## 1. 確定事項

1. ProviderはSync Agent設定で`Labonity`または`Liberty`を明示選択する。
2. 両Providerとも入口設定ファイルまでの完全パスをSync Agent設定に保持する。
3. Labonityは`LibLocal.xml`から`LibertyDatabaseSetting.xml`を解決する。
4. Libertyは`LIBLocal.Ini`から`LIBCTRL.ini`を解決する。
5. OCR同期の登録先は両Providerとも出荷DB内の`FieldTest`スキーマである。
6. Libertyでも既存フレッシュテーブルへは書き込まない。
7. 品質系DBは第1.0版OCR同期では使用しない。
8. Providerアプリケーションは`FieldTest`スキーマを参照して結果を利用する。

## 2. 構成方針

Sync Agentは、Provider設定解決、既存DB読取、DTO変換、FieldTest登録、クラウド同期に責務を限定する。Providerアプリケーションの表示・帳票・参照実装はProviderアプリケーション側の責務である。
