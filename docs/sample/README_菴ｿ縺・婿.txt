現場試験アプリ OCR黒板対応サンプル
====================================

使い方
------
1. ZIPを解凍します。
2. labonity_field_test_ocr_mock_v3/index.html をブラウザで開きます。
3. サーバー起動や npm install は不要です。

主な操作
--------
- ログイン画面の「ログイン」を押すと予定一覧に進みます。
- 予定一覧で「開く」を押すと出荷予定詳細へ進みます。
- 出荷実績の「開く」を押すと出荷実績詳細へ進みます。
- 出荷実績詳細の「写真を追加」を押すと写真追加メニューが開きます。
- 写真追加メニューには以下の3つがあります。
  - カメラで撮影
  - OCR取込用黒板を撮影
  - ライブラリから選択
- 「OCR取込用黒板を撮影」を選ぶと「OCR黒板写真の確認」画面へ進みます。
- 「OCR取込用に保存する」を押すと保存完了画面で「OCR予約済」「同期済」が確認できます。
- 「写真一覧を見る」では、OCR黒板バッジ、OCR済/予約済/対象外、信頼度を確認できます。
- 写真のサムネイルを押すと写真詳細を表示できます。
- 右上の「更新」はサンプルデータを初期状態に戻します。

実装内容
--------
- 添付モックアップの8画面に近いスマホUIとして作り直しています。
- iPhone風フレーム、ステータスバー、カード、ボトムシート、バッジ、下部ボタンの見た目を寄せています。
- 実API、Liberty Account、Blob Storage、OCRエンジンは呼び出していません。
- 写真・OCR状態・同期状態はブラウザ内のサンプルデータで動作します。

対応した変更点
--------------
- 写真追加メニューに「OCR取込用黒板を撮影」を追加。
- 出荷実績詳細に「OCR黒板」「OCR状態」「信頼度」「同期状態」を追加。
- 写真確認画面を「OCR黒板写真の確認」に対応。
- 写真一覧に「OCR黒板」バッジと「OCR済・信頼度」「OCR予約済」「OCR対象外」を追加。
- 保存完了画面に「OCR予約済」「同期済」を追加。
- 予定一覧・出荷実績一覧にも OCR 状態の補助バッジを追加。


仕様メモ:
- 1つの出荷に対して、画面上で有効なOCR取込用黒板は1件だけとして扱います。
- OCR黒板を追加撮影した場合は差し替え扱いとし、既存のOCR黒板写真は通常写真扱いに戻します。
- そのため同一出荷のサマリーに「OCR済」と「OCR予約済」が同時に出ないようにしています。
- 「同期済」は写真保存・メタデータ同期の状態であり、「OCR予約済」「OCR済」とは別軸の状態です。

更新 v5
- OCR取込用黒板写真のサンプルを、作業員・試験器具を含まない「黒板のみを撮影した画像」に差し替えました。
- 出荷実績詳細、OCR黒板写真確認、写真一覧に表示されるOCR黒板サムネイルも黒板単体画像に統一しました。
- 通常写真は現場状況写真のまま残し、OCR取込用黒板写真と区別できるようにしています。
